# 沐寧 · 用藥提醒 資料流設計（2026-07-03 · Edward 點題後定）

> Edward：「用藥這件事要想清楚邏輯怎麼串——用戶怎麼告訴系統什麼時間用什麼藥、要用多久。」
> 紅線不變：**寧寧是鬧鐘、不是藥師**——不建議藥、不談劑量調整、不判斷交互作用；藥怎麼吃聽醫生的。

## 一、藥的資料怎麼進系統（三條路）
1. **家人／本人在 App 設定（V1 主路、原型已做）**：設定 → 提醒管理 → 新增：藥名（照藥袋抄）＋時間（可註記飯後/睡前）＋吃多久（N 天或長期）。誰設的會記名（美華設定／阿嬤設定）——對應家人帳號的照護者權限。
2. **聊天中順手建檔（V1.5、接語音主線）**：長輩跟寧寧說「醫生開了新藥」→ 寧寧引導問三件事（叫什麼名字？一天什麼時候吃？吃多久？）→ 產生**提醒草稿** → 通知家人確認後生效（防聽錯藥名）。
3. **拍藥袋自動帶入（V2 backlog）**：拍照識別藥袋上的藥名/用法 → 草稿 → 家人確認。台灣藥袋格式固定、可行性高。

## 二、一筆藥的格式
`{ 藥名, 時段[]（每次時間＋飯後/睡前註記）, 開始日, 天數|長期, 設定人, 給誰 }`
→ 引擎側對應 routine_reminders（表已存在）；原型暫存本機。

## 三、這筆資料會流去哪（閉環）
1. **每天的任務**：自動長進「今天一起完成」（到期自動消失——90 天吃完就不再出現）。
2. **到點提醒**：寧寧語音喊（用藥提醒畫面：大藥名＋「我吃過了」／「等 10 分鐘再提醒」兩大鈕）。
3. **記錄與存摺**：吃了→安心存摺累積；家人端看得到準時率。
4. **漏服**：超時未確認 → 提醒一次 → 仍沒回應 → 通知家人（走安全網、不是罵人）。
5. **停藥/改時間**：設定裡改；聊天中說「醫生說停了」→ 寧寧產停用草稿 → 家人確認。

## 四、原型現況（2026-07-03）
✅ 設定 → 提醒管理面板：列現有藥（含誰設定）＋新增表單（藥名/時間/吃多久）＋「看看提醒時長輩會看到什麼」預覽（用藥畫面終於有入口）＋返回。存本機。
⏳ 接任務清單自動生成、到點觸發、漏服通知＝家人帳號 P0 之後接引擎。
