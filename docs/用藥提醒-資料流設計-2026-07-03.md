# 沐寧 · 用藥提醒 資料流設計（2026-07-03 · Edward 點題後定）

> Edward：「用藥這件事要想清楚邏輯怎麼串——用戶怎麼告訴系統什麼時間用什麼藥、要用多久。」
> 紅線不變：**寧寧是鬧鐘、不是藥師**——不建議藥、不談劑量調整、不判斷交互作用；藥怎麼吃聽醫生的。

## 一、藥的資料怎麼進系統（三條路）
1. **家人／本人在 App 設定（V1 主路、原型已做）**：設定 → 提醒管理 → 新增：藥名（照藥袋抄）＋時間（可註記飯後/睡前）＋吃多久（N 天或長期）。誰設的會記名（美華設定／阿嬤設定）——對應家人帳號的照護者權限。
2. **聊天中順手建檔（V1.5、接語音主線）**：長輩跟寧寧說「醫生開了新藥」→ 寧寧引導問三件事（叫什麼名字？一天什麼時候吃？吃多久？）→ 產生**提醒草稿** → 通知家人確認後生效（防聽錯藥名）。
3. **拍藥袋自動帶入（V2 backlog）**：拍照識別藥袋上的藥名/用法 → 草稿 → 家人確認。台灣藥袋格式固定、可行性高。

## 二、一筆藥的格式
`{ 藥名, 時段[]（每次時間＋飯後/睡前註記）, 開始日, 天數|長期, 設定人, 給誰 }`
→ 引擎側對應 routine_reminders（表已存在）；原型暫存本機。

## 三、這筆資料會流去哪（閉環）
1. **每天的任務**：自動長進「今天一起完成」（到期自動消失——90 天吃完就不再出現）。
2. **到點提醒**：寧寧語音喊（用藥提醒畫面：大藥名＋「我吃過了」／「等 10 分鐘再提醒」兩大鈕）。
3. **記錄與存摺**：吃了→安心存摺累積；家人端看得到準時率。
4. **漏服**：超時未確認 → 提醒一次 → 仍沒回應 → 通知家人（走安全網、不是罵人）。
5. **停藥/改時間**：設定裡改；聊天中說「醫生說停了」→ 寧寧產停用草稿 → 家人確認。

## 三之二、設定入口與操作（Edward 2026-07-03 拍板：兩個入口＋越少打字越好）
- **兩個入口**：①狀態頁（四格數據下方「用藥提醒 · N 種藥 ›」列——看健康時順手管藥）②設定頁「提醒管理」。同一個面板。
- **操作原則＝用點的、不用打字**：只有藥名要打（照藥袋抄）；「什麼時候吃」點時段籤（早餐後/午餐後/晚餐後/睡前、可多選、各帶預設時間）；「吃多久」點 7 天/30 天/90 天/長期。兩下點完、寧寧口頭複誦確認。
- 之後長輩語音路（②）同樣三問就好：藥叫什麼？什麼時候吃？吃多久？——跟畫面同一套心智模型。

## 四、原型現況（2026-07-03）
✅ 設定 → 提醒管理面板：列現有藥（含誰設定）＋新增表單（藥名/時間/吃多久）＋「看看提醒時長輩會看到什麼」預覽（用藥畫面終於有入口）＋返回。存本機。
⏳ 接任務清單自動生成、到點觸發、漏服通知＝家人帳號 P0 之後接引擎。

## 補充（2026-07-03 晚 · Edward 拍板）
- 設定頁入口移至「健康資料與安全」區（原在 App 體驗，歸類錯誤）。健康區順序：Apple 健康 → 用藥提醒 → 看診提醒 → 安全通知 → 隱私與資料。
- 新增「看診提醒」lite（原型已可用）：14 天快選＋上午/下午/晚上 → 存回診日；寧寧前一天提醒＋回診摘要先準備好。正式版接引擎排程與回診摘要生成。
- 設定頁整體資訊架構重排：上=人（帳號/訂閱/家庭/陪伴角色）、中=健康與安全、下=App 本身（體驗/關於/免責）。品牌卡移至底部「關於」。

## 相對時段制（2026-07-03 Edward 拍板：不寫死幾點——每個人作息不同）
- 用藥時段存「相對錨點」：早餐後／午餐後／晚餐後／睡前（不含固定時刻）。
- 實際提醒時間 = 本人作息表的該餐時間 + 30 分（睡前 = 就寢前 30 分）。
- 個人作息表（早餐/午餐/晚餐/就寢）來源：①首次設定時寧寧聊天裡順口問 ②個人資料頁可改 ③未知前用預設 08:00/12:30/18:30/21:30，並在聊天中主動確認修正。
- 畫面一律只顯示「早餐後」等字樣；面板加一行「提醒時間跟著你的作息走」。
