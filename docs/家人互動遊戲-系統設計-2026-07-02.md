# 沐寧 · 家人互動遊戲 系統設計（2026-07-02）

> Edward 需求：互動遊戲選單 → 選遊戲 → 找人加入 → 等人到齊/約時間 → 開始；例：一天內全家集體走幾步。含獎勵成就設計。
> 依據：蕪菁頭 2026-07 調研（微信運動/台灣 Walkr/Pikmin Bloom 週挑戰/Apple 手錶競賽/日本健康點數 4 年實證/國泰健走/ElliQ 語音遊戲/Wordle 家庭儀式/Forest 共同種樹/法國 Famileo 實體家庭報…全附來源於調研原稿）＋現有 App 骨架（發起挑戰面板已有「選類型＋勾人＋依能力份額」雛形）。
> 視覺一律採 `UIUX-重設計` 定案語彙（munea.net 色系＋心情圖譜 v2）。

## 一、設計原則（每條有市場屍體或實證背書）
1. **合作優先、競爭軟性可選**——75+ 研究：長輩玩的動機是社交、迴避競爭。預設「全家 vs 目標」，不是「家人 vs 家人」。
2. **同一目標、各自標準（不對稱貢獻）**——照 Apple「個人目標百分比」計分：**阿嬤走 3,000 步（她的 100%）跟兒子走 12,000 步貢獻等值**。現有 UI 的「份額」概念保留、升級成百分比制。
3. **不用湊人在線**——照 Pikmin 週挑戰：**報名窗口取代等候室**、到點自動開始、可中途補位。忙碌子女只能這樣玩。
4. **長輩語音一句話入局**——寧寧問「美華邀你今晚玩猜歌，要加入嗎？」答「好」＝完成報名（ElliQ 已驗證這條路長輩走得順）。長輩不點選單。
5. **每日重置的溫柔節奏**——不做懲罰性連勝（會焦慮棄用）；斷了可修復、或學微信運動每天歸零重來。
6. **數據全自動**——日本 4 年實證：要手動上傳的那群人健康效果＝0。步數走 Apple 健康自動帶入（App 已接）。
7. **通知極簡**——成人端一天最多 1 則遊戲摘要；邀請走 App 推播＋LINE 分享卡（台灣 65+ 91.5% 用 LINE）。
8. **社交層自己養**——Fitbit/Google 都砍了健康社交功能（大反彈）＝市場真空；我們的遊戲不必自己賺錢，是掛在照護關係上的黏著層。

## 一之二、Edward 修正（2026-07-02 深夜 · 憲法級邊界）
**「健康作息不做家庭挑戰」**：早點睡／量血壓／用藥全勤**移出**家庭挑戰（每個年齡與身分的生活作息不同，家庭活動不過度影響彼此生活步調）——這些仍是**個人的**「今天一起完成」任務與提醒，只是不拿來當全家比賽。
家庭互動三支柱（發起面板已照改、原型已實作）：
1. **一起運動挑戰**（走路等；百分比份額公平制不變）
2. **機智問答小遊戲**（家族大考驗/猜歌系列）
3. **揪一攤（活動投票）⭐ 新核心功能**：發起聚餐/出遊 → 填「要揪什麼＋哪一天」→ 勾家人 → 寧寧親口問長輩、其他人 App 回「去/沒空」→ 回覆狀態一眼看（去=薄荷、沒空=灰、等回覆=珊瑚橘）→ 回覆齊了通知發起人。例：「週六晚上全家聚餐 7/6」。這是把 App 從「關心工具」變成「家庭聚會發生器」的功能——長輩最期待的就是相聚。

## 二、開局流程（定案 · 對應 Edward 的 flow）
```
互動遊戲選單（家人分頁「一起玩」區）
 → 選遊戲（動一動/動腦/回憶 三類卡片）
 → 選時間（今晚 8 點｜週日下午｜「週一自動開始」的週挑戰）
 → 勾家人送邀請（App 推播＋LINE 分享卡；長輩端＝寧寧語音邀請）
 → 報名窗口頁（取代等候室）：已加入頭像＋暖語「阿嬤已就位」＋倒數
    · 不強制到齊：到點 ≥2 人自動開始、晚到可補位
 → 進行中：進度頁看全家一起前進（寧寧當啦啦隊播報）
 → 結算：慶祝畫面 → 獎勵入帳 → 自動寫進家庭記錄簿
兜底節奏：每週一自動生成「本週全家挑戰」——沒人發起也永遠有局在跑（Wordle/微信運動長青的根本）
```

## 四、獎勵與成就（三層）
1. **個人層（私密、溫柔）**：只有「完成」沒有「排名」；里程碑（「這個月你陪家人玩了 12 次」）；連續紀錄可用「補牆卡」修復、絕不懲罰斷鏈；**長輩端永不顯示輸/落後**。
2. **家庭集體層（公開、視覺化）**：全家花園開花；家庭成就簿（「連續 4 週全家達標」「三代同場玩過猜歌」——**成就主體是這個家**）；要比只比「這一家 vs 上週的自己」。
3. **真實世界層（最關鍵差異化）**：點數換的不是虛寶，是**家庭行動**——月度**實體照片故事報**（Famileo 模式、可做加值訂閱）、「全家晚餐之約」（達標後寧寧幫忙喬時間）、寧寧安排的視訊之約。日本實證：小額實質回饋就足以產生 4 年期健康行為改變。中長期 BD 口：台灣保險外溢（國泰健走模式）/縣市健康點數。
4. **防退坑紅線**：❌ 懲罰性連勝 ❌ 長輩公開排行榜 ❌ 幼稚化貼紙轟炸（獎勵語言像「家人間的感謝」不是給小孩的）❌ 通知轟炸 ❌ 手動上傳 ❌ 步數無上限（設日認列上限、防刷又保護走得慢的人——學台北 U-Walk）。

## 五、工程落點（誠實）
- **依賴順序**：遊戲需要「家人是真的」→ **排在家人帳號連動 P0 之後**（邀請/成員/份額都吃帳號架構）。
- 資料：挑戰主檔＋參與者＋進度事件；互動事件用既有 `family_interaction_events`（invite/accept type 已在）；新表（challenges）DDL 與 Codex 對齊後加。
- 步數：Apple 健康自動同步（App 已接線）；進度聚合走背景整理（跟感知每日排程同管線）。
- 寧寧角色：邀請唸稿/報名口答/啦啦隊播報/主持問答——全部走既有主動開口＋語音管線，**零新模型**。
- 順位理由：M1 兩款＝一週節奏＋一日節奏、覆蓋「動身體＋動腦」、工程最省；M2 語音含金量最高（人設放大器）；M3 內容飛輪＋獎勵層；M4 重工程壓後。

## 七、活動系統 v2（2026-07-03 · Edward 走查後定案）

### Edward 拍板（全部有效）
1. 名稱統一「**發起活動**」（不叫發起挑戰）。
2. **一起運動**：發起人用**拉桿**設總目標步數（10,000–60,000、每格 5,000）＋期間（3 天／一週）；邀請名單**不顯示份額**（能力分配是寧寧內部的事，不給家人看數字比較）；進行中卡片要看得到**每個人各走了多少**（貢獻籤）。
3. **機智問答**：拉桿設題數（5–20、每格 5）；結束後看**排行榜**（第幾名＋答對幾題）；長輩「用說的就能玩」、其他人手機作答。
4. **揪一攤**：日期**必須是系統吃得到的結構化選擇**（今天／明天／這週六／這週日／選日期＋時段），不是自由文字；**過了活動日，卡片自動收進家庭記錄簿、從牆上消失**，家庭動態補一句。

### 活動生命週期（三種通用）
```
發起（設定目標/日期）→ 邀請（寧寧親口問長輩、其他人 app 通知）
→ 進行中（卡片：運動=總進度+每人貢獻；問答=誰答到第幾題；揪一攤=出席看板）
→ 結束（時間到 / 目標達成 / 活動日過）
→ 結算卡自動入家庭記錄簿＋家庭動態一句話（運動=達標慶祝；問答=排行榜；揪一攤=出席紀錄）
```

### 各類型欄位（正式版資料）
| 類型 | 發起時設定 | 進行中顯示 | 結束結算 |
|---|---|---|---|
| 一起運動 | 目標步數(拉桿)、期間(3天/一週)、成員 | 總進度條＋每人步數籤 | 達標/未達、每人貢獻，入記錄簿 |
| 機智問答 | 題數(拉桿)、成員 | 誰答到第幾題 | 排行榜(名次+答對數)，入記錄簿 |
| 揪一攤 | 名稱、日期(結構化)、時段、成員 | 出席看板(能到/沒空/等回覆) | 出席紀錄，活動日隔天自動收卡 |

### 原型已做（web/，2026-07-03）
- 發起面板三類型各自的設定區（拉桿×2、日期時段籤）；運動模式隱藏份額。
- 發起後活動存在手機裡（正式版換雲端），重開 app 卡片還在。
- 過期自動收卡＋家庭動態補一句（已實測：過期活動重開後消失、動態出現結束訊息）。
- 問答結束示範卡：金/銀/銅名次＋答對題數＋阿嬤用說的作答註記。

### 正式版待接（引擎）
- 活動表(family_activities)入雲端、跨裝置同步——依 007 資料表協調規則與 Codex 對齊後才動格式。
- 步數來源：Apple 健康/穿戴 → 每人貢獻即時更新。
- 問答題庫＋寧寧主持（語音出題、語音作答判定）。
- 寧寧邀請流程：長輩端下次聊天親口問「美華揪你週六聚餐，去嗎？」→ 回答直接寫回出席看板。

## 八、日期選擇與問答運作（2026-07-03 補）

### 日期模組決策（Edward 問「用手機原生還是自己設計」）
- **V1 定案：自家設計的 14 天快選格**（7 欄×2 列，格子＝星期＋日期，今天/明天有字樣，選中滿版品牌綠）。
- 為什麼不用瀏覽器/手機原生日期格：長得像表單欄位、跟整個 app 的點選籤語言不合、長輩要點小小的年月日很吃力；家庭聚會 9 成約在兩週內，14 天快選涵蓋主要情境。
- 之後包成正式 app 時再評估：超過 14 天的遠期活動，V2 可加整月月曆（仍是自家樣式，不用原生）。

### 機智問答完整運作（不是假頁面——原型已可作答）
1. **發起**：選題數（拉桿 5-20）＋找人 → 出一張活動卡。
2. **作答（各自來、不用同時在線）**：發起人點卡片直接作答（原型已做：一題一畫面、四選一、點了立刻顯示對錯、自動下一題）；其他家人收到通知後在自己手機作答；**長輩不用碰按鈕**——寧寧下次聊天時親口出題、聽語音回答記分。
3. **進行中卡片**：顯示誰答完了（「你答對 4/5——等阿嬤、美華作答」，原型已做）。
4. **全員答完**：卡片翻成排行榜（金/銀/銅＋答對題數，樣式已上線），入家庭記錄簿，家庭動態推一句。
5. **題庫**：原型內建 5 題示範；正式版由寧寧依家庭興趣出題（引擎待接），難度自動照年齡調。

### 提示視窗規則（2026-07-03 Edward 拍板）
- 通話頁的提示（如「字幕已開啟」）一律出現在**按鈕上方**，不得蓋住任何可點的東西。

## 九、結果卡收版規則（2026-07-03 · Edward「已結束卡不能一直佔版面、但結果要大家看過才有意義」後定案）

1. **看過 = 收版條件**：每位成員第一次看到結果（打開家人頁看到排行/達標卡）記「已看過」；**長輩由寧寧下次聊天親口播報結果，播報完即算看過**——不逼長輩自己滑。
2. **全員看過 → 隔天收進家庭記錄簿**（牆上消失、記錄簿永久保存、家庭動態補一句）。
3. **有人一直沒看 → 最多停 3 天**：第 3 天照收，寧寧主動告訴沒看到的人結果。
4. **牆上同時最多一張已結束卡**：新的結束卡出現時，舊的直接進記錄簿。
5. 卡片腳註隨狀態變：「等大家都看過排名就收進記錄簿 · 最多留 3 天」→ 全員看過改「大家都看過了，明天收進記錄簿」。

原型已做：答完的問答卡翻成「已結束」樣態（不能重複作答）＋滿 3 天自動收進記錄簿＋動態補一句；「誰看過」的即時追蹤是正式版引擎項目。

## 十、發起面板 v3 ＋ 文案原則（2026-07-03 Edward 拍板）

1. **文案哲學（全 app 通用）**：不催用戶「來聊聊」。問候句只關心人（吃飽了嗎／睡得好嗎）；陪伴卡上那行顯示「她記得的事」（記憶回呼，例：昨天說到孫子快畢業——她都記得）；行動入口只有按鈕本身，不重複推。連續天數這種「打卡壓力」字眼不出現在用戶面前。
2. **一起運動**：天數改拉桿 1–7 天；目標步數隨「人數 × 天數」自動建議（基準每人每天約 4,000 步、範圍每人每天約 1,500–8,000），改人數或天數即時重算，拉桿下一行寫「N 人一起走 D 天 · 平均每人每天約 X 步」。
3. **發起面板 Google 邀請式**：邀請從四大列收成一排四顆頭像籤（點選、勾號徽章）；逐列小字改為一行說明（隨類型換：阿嬤那份寧寧親口問／用說的就能玩／回去沒空）；目標說明框刪除。整張面板一屏內完成。
4. **點數字色統一白色**（通話待機膠囊）。
