## 一句話
使用者說話 → **指揮層**按優先權叫出〔人格＋記憶＋感知＋安全〕組成脈絡 → **語音層**即時回覆 → **會動的臉**同步 → 事後**記憶層**整理。
公式：**回應 = 角色人格 ＋ 記憶 ＋ 感知 ＋ 當下對話 ＋ 安全規則 ＋ 語音/臉表達限制。**

---

## A. 五個「可換／可調研」技術層

### 1. 語音層（即時 S2S）
- **職責**：即時聽與說、可打斷、語音臨場感。
- **固定介面**：前端 `MuneaVoiceProvider`＋後端即時語音橋（WebSocket）；合約＝開 session／送麥克風音／回語音／打斷。
- **可換引擎**：Gemini 3.1 Flash Live（現用）｜OpenAI Realtime｜LiveKit／Pipecat（拿框架級打斷）｜自架 Moshi/Ultravox。
- **現況**：🟢 Gemini Live 已串接可用（`engine/live_voice_server.py`）。
- **調研/迭代**：延遲、打斷精度、台語品質、供應商退路。

### 2. 記憶層
- **職責**：記得你——四層（核心/長期/近況/當天）＋重要分數＋整理員。
- **固定介面**：`/memory/extract`（存）、`/memory/retrieve`（取）、`/butler/post-turn`（聊完整理）；`memory_items` 格式（含 `valid_from/valid_until/supersedes` 時序欄位）。
- **可換底層**：Supabase Postgres + **pgvector**（主路）｜Mem0｜Zep-Graphiti（借鏡做法、非依賴）。
- **現況**：🟡 骨架在＋真萃取剛做（`engine/memory_engine.py`，只存長輩事實）；pgvector／四層／整理員／收斂舊側寫待做。
- **調研/迭代**：召回準度、時序衝突、遺忘衰減。

### 3. 感知層
- **職責**：懂此刻——時間/天氣/在地/家人狀態/興趣＋每日清晨「今日簡報」。
- **固定介面**：`/perception/topic-plan`（判斷要哪些源）、`/perception/snapshot`（存取事實，帶 `expiresAt`）；工具呼叫（function calling）。
- **可換來源**：中央氣象署 CWA｜Google Maps grounding／MCP｜行事曆。
- **現況**：🔴 計畫抽象在，但真來源 0 接、天氣寫死。
- **調研/迭代**：grounding 來源、MCP、每日排程。

### 4. 會動的臉層（Avatar）
- **職責**：表情／嘴型／臨場，同步語音。
- **固定介面**：前端 `MuneaAvatarRuntime`（setState/setCharacter/speak）＋後端 `/avatar-session`（選 provider＋權益）。
- **可換引擎**：2D viseme（保底）｜Ditto（自架）｜HeyGen LiveAvatar／Tavus／Anam（雲端 API）｜NVIDIA Audio2Face-3D。
- **現況**：🟡 2D 假動在；真臉未接、`/avatar-session` 後端待補。
- **調研/迭代**：fps／成本／授權；**先用雲端 API 驗留存，再決定自架**。

### 5. 安全層（守護）
- **職責**：危機紅線（自傷/急症/醫療邊界）——**可否決上面任何一層**。
- **固定介面**：`/guardian/evaluate`（風險等級＋動作）；非機率獨大，規則優先。
- **可換/可補**：決定論關鍵字（現）＋語義分類器（待）＋ moderation。
- **現況**：🟢 決定論＋中英危機詞庫＋20 題測試；🟡 語義分類器待加（補隱晦憂鬱/失智/跌倒召回）。
- **調研/迭代**：召回率、雙層（確定性＋語義）。

## B. 兩個「我們自己的」層（不外求、是招牌）

### 6. 角色人格層
- **職責**：六角色的個性/語氣/關係——**換任何引擎，寧寧還是寧寧**。
- **固定介面**：`/persona/context`；`characters.json` ＋ `PERSONA_TEMPLATES` ＋ `companion_relationship_states`（關係成長）。
- **現況**：🟢 六角色設定＋聲線＋關係狀態齊。
- **性質**：我們的 IP／設定，不是去外面換的技術。

### 7. 指揮層（調度 · 即調研裡的「多腦協同」）
- **職責**：把五層串成一句回應；**定優先權（安全 ＞ 即時語音 ＞ 背景管家）**；context **最小 scoped 注入**（不爆 token）。
- **固定介面**：`build_reply_context` ＋ `reply_context_instruction`（未來抽成明確指揮/router）。
- **現況**：🟡 組裝已在，缺「顯式優先權仲裁」；待補。
- **性質**：我們手刻的協同，不引入重框架（即時對延遲敏感）。

## 分層原則（為什麼這樣切）
1. 每層對著外面一個技術市場 → **要調研哪層盯那層、要換就換那層，不動別層。**
2. 安全獨立成層、能否決一切——不跟功能混在一起被換掉。
3. 人格獨立＝差異化不被引擎綁架。
4. 指揮層負責優先權與最小注入——解掉調研點名的「無 router／context 爆量」風險。
