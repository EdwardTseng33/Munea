## 5 條防打架鐵律（輕量看板模式）

1. **動工前先登記**：在上表寫下你要碰哪些檔，掃一眼對方在動什麼、避開。
2. **動工前先 `git pull`**：永遠在最新版上疊，不要在舊版上改。
3. **小步存、存完馬上上傳**：縮短兩邊分岔的時間窗，git 會自動把「改到不同檔」的合起來。
4. **絕不覆蓋式上傳（force push / `-f`）**：永遠不蓋掉對方的存檔。萬一上傳被擋（non-fast-forward）= 對方剛上傳過 → 先 `git pull` 併進來再上傳。
5. **守好自己的 lane**：非改共用檔（`server.py` 路由、`00-總綱`）不可時，先在看板喊、改完馬上上傳、格子標回空。

> 衝突了怎麼辦：git 會標出打架的檔。**不要硬蓋**——貼到看板、由城堡（整合角色）收乾淨。
