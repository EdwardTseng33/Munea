# RTX 5090 真 768 容量測試設計（2026-07-13）

## 目的

確認 RunPod RTX 5090 在沐寧現役 SoulX-FlashHead Lite 真 768 輸出下，能否穩定承載 2 人或 3 人同時生成。這是獨立 GPU 容量驗收，不切正式 Avatar，不修改 App、iOS、Cloud Run、Gateway 或 Claude 的高併發工作。

## 固定基準

- 模型：SoulX-FlashHead Lite，來源固定在 commit `9bc03de39e1363b18e587d29b286708e1174dba9`。
- 解析度：模型直接輸出 `768x768x3`，不得先算低解析度再放大。
- 時間單位：25fps、每塊 24 幀，即每塊必須在 960ms 內完成。
- 角色：a05 擬真女性與 a06 擬真男性都要跑。
- 音檔：沿用 `poc-mandarin.wav`，避免內容長度與音訊特徵造成偏差。
- 模式：先 eager 驗單路，再以 compile 做正式容量與 120 塊長時間穩定測試。

## 通過門檻

1. 每一塊輸出都必須是 `768x768x3`，過程不可跳尺寸、不可 OOM。
2. 硬即時門檻是 p95 不超過 960ms；正式上線門檻是 p95 不超過 768ms，保留至少 20% 突發餘裕。
3. 120 塊長測不可逐步變慢，最大值、顯存峰值、GPU 使用率與首次暖機時間都要留下紀錄。
4. 2 路未過就停止 3 路；3 路未保留 20% 餘裕，就不再測 4 路。

## 執行順序

1. 開卡後先以 `nvidia-smi` 核對確實是 RTX 5090，記錄驅動、CUDA、顯存與每小時價格。
2. a05、a06 各跑單路 eager，確認模型、權重、角色圖與真 768 都正確。
3. 單路 compile 暖機後跑 120 塊，取得穩態基線。
4. 同時跑 2 路 compile；通過正式門檻後才測 3 路。
5. 依同一規則換另一角色複核，避免只對單一角色最佳化。
6. 匯出結果後立即 terminate Pod，重新查清單確認沒有測試卡仍在計費。

## 成本判讀

以畫面價格 US$0.99/hr、US$1 = NT$32 試算：

| 可保證同時生成 | GPU 每人每分鐘 |
|---:|---:|
| 1 人 | 約 NT$0.528 |
| 2 人 | 約 NT$0.264 |
| 3 人 | 約 NT$0.176 |

因此 RTX 5090 至少要安全承載 2 人才追平 RTX 4090 單人真 768 的每人成本；能安全承載 3 人，才有明顯的成本與容量優勢。

## 費用與安全護欄

- 本輪最高支出 US$5、最長 2 小時；任一條先到即停止。
- 不建立 Snapshot、不掛網路磁碟、不啟動正式流量。
- API 金鑰只由執行程序環境變數帶入，不寫入 repo、文件或測試結果。
- 無論成功、失敗或安裝中斷，最後都必須 terminate，並以 RunPod 清單再次確認。

## 本測試不代表的事情

GPU 算力通過不等於通話體驗已通過。嘴聲同步、句尾截斷、麥克風收音與開場打招呼仍需在完整 Voice + Avatar 同線及 iPhone 真機另外驗收。
