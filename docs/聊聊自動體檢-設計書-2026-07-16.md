> Edward 7/16 交辦：「每個之前一直有問題的地方，通過/沒通過，你要想辦法測到。」
> 目的：把「Edward 用耳朵抓問題」變成**一鍵自動體檢**——每項給 ✅ 通過／❌ 沒通過＋證據檔；

**真人保留一關**：整體體感（自然度、溫度）——機器排除掉硬故障後，Edward 只驗「像不像人」，不用再當人肉除錯器。

## 二、跑法與掛鉤

```
npm run health:chat -- --target staging          # 測試機
npm run health:chat -- --target prod             # 正式
npm run health:chat -- --target <canary網址>     # 晾著的新版
```
- 產出：一張 通過/沒通過 總表＋證據檔（音檔、逐字、時序）存 `artifacts/chat-health/<日期時間>/`
- **掛鉤（硬規矩）**：`promote.sh`（測試機切流量）與 `prod-deploy.sh`（正式出新版）執行前自動跑一期全自動項，**紅燈直接擋下**
- 帳號：專用測試帳號（不污染真資料；記憶測試每輪自動清）
