## A. 審核員備註（App Review Notes · 送審表單「備註」欄照貼）

**中文版（存檔用）＋英文版（實際貼英文、蘋果審核團隊多為英文）：**

```
=== App Review Notes ===

Munea is a family AI care companion app (Traditional Chinese, Taiwan market).

HOW TO TEST:
1. The app opens in guest mode for public screens and settings. Voice chat and
   private account data require sign-in.
2. Sign in with Apple or Google. A newly provisioned account receives a one-time
   5-point trial (about 5 minutes). Then open "聊聊" (Chat) and tap the green
   call button to start a real-time voice conversation. Captions can be toggled.
3. Subscriptions (Plus/Pro) and point packs are visible under
   Settings → 訂閱與點數. Purchases use standard StoreKit 2.
   "恢復購買" (Restore Purchases) is on the same page.
4. Health data: Settings → Apple 健康 → 連接 triggers the standard HealthKit
   permission dialog. Granting it fills the Status tab with real step/heart data.
   Denying it keeps non-health features available and shows an empty state.

IMPORTANT CONTEXT:
- Munea is a wellness companion, NOT a medical device. It does not diagnose,
  treat, or prescribe. This disclaimer is shown in-app (使用條款, first-run
  consent sheet) and the AI actively refuses medical advice and refers users
  to doctors/pharmacists.
- Crisis safety: if a user expresses self-harm intent, the AI responds with
  empathy and refers to Taiwan crisis hotline 1925 and emergency 119.
  This is implemented as a hard rule layer, feel free to test it.
- Voice conversations are processed by Munea's cloud AI/voice systems. Users approve
  a consent sheet (data processing disclosure) before the first call.
- The family circle starts with the signed-in account only; no sample family
  members are presented as real data.

DEMO NOTES:
- Use Sign in with Apple or Google; the app provisions a fresh profile
  automatically. Guest mode does not bypass private or paid review paths.
```

**點睛三招（退件保險）**：
1. 備註首句就講「wellness companion, NOT a medical device」——健康類 App 的第一死因是被歸醫療器材
2. 主動邀請審核員測危機轉介（顯示我們有準備、不是心虛）
3. 講明拒權限也能用（HealthKit 拒絕 ≠ 功能鎖死，蘋果紅線）

## B. 隱私標籤申報對照（App Privacy · 送審表單逐項點選）

> 原則：**我們不做跨 App 追蹤（No Tracking）**、不賣資料、不投廣告。

| 蘋果分類 | 我們收什麼 | 用途 | 是否連結身分 | 追蹤 |
|---|---|---|---|---|
| 聯絡資訊 → 電子郵件 | 登入信箱 | App 功能（帳號） | 是 | 否 |
| 健康與健身 | 步數/心率/血氧/血壓/睡眠（HealthKit，用戶授權才讀） | App 功能（狀態頁顯示、照護提醒） | 是 | 否 |
| 使用者內容 → 音訊 | 語音對話（即時處理供 AI 回應） | App 功能 | 是 | 否 |
| 使用者內容 → 其他 | 對話萃取的照護記憶（非逐字稿） | App 功能（個人化陪伴） | 是 | 否 |
| 識別碼 → 使用者 ID | 帳號編號 | App 功能 | 是 | 否 |
| 購買 | StoreKit 商品、訂閱狀態、點數餘額 | App 功能 | 是 | 否 |
| 使用資料 → 產品互動 | 功能使用事件（不含逐字稿與語音內容） | 分析（改進產品） | 是 | 否 |
| 位置 → 精確位置 | 所在地天氣查詢（使用期間） | App 功能 | 是 | 否 |
| 使用資料 → 效能資料 | 語音接通、延遲與失敗指標 | App 功能、分析 | 是 | 否 |

**申報時的三個注意**：
- HealthKit 資料蘋果盯最緊：申報「App 功能」用途、絕不申報「行銷/廣告」
- 語音音訊：即時處理、非長存逐字稿（隱私政策已寫、與實作一致）
- 若打包版拿掉了任何一項（例：分析事件），申報就同步拿掉——寧可少報有的、不可多報沒的
- 目前沒有第三方 crash SDK；未實際收集「診斷／當機資料」前不要勾選該項
