## ✅ 核心已解法：完整照片比例正確顯示在手機上（這是定案、照做）

**架構＝三層，不要再改成別的：**
1. **全身立繪（背景層）＝ 9:16 完整照片鋪滿手機**
   - 檔：`web/flashhead/bg-a05.png`（女）/`bg-a06.png`（男），1080×1920
   - CSS：`.fh-frame { width: max(100%, calc(100dvh*1080/1920)); aspect-ratio:1080/1920 }`＋`.fh-bg { object-fit:cover; inset:0 }`
   - **滿版無黑邊關鍵＝`max(100%,…)`**（不是 min；min 會留黑邊，Edward 7/11 點名）
   - **立繪必須永遠 opacity:1**：`.face-bg.livevid img{opacity:0}` 這條要寫成 `img:not(.fh-bg)`，否則立繪被一起藏→露深綠底＝「墨綠遮罩」（幾版都沒抓到的真兇，7/11 才解）
   - 頂部**不要漸層紗**：`.face-topscrim{background:none}`（照片乾淨頂到邊）
2. **動臉（模型層）＝只動頭/頭肩區，貼回立繪臉部位置**
   - 模型輸出 512×512（**壓成正方形是模型工作格式、不是給人看的**）
   - overlay 框 `.fh-overlay { top:7.2917%; height:75%; width:100% }`＋`video{object-fit:fill}` → 把 512² 拉回 3:4 貼到立繪臉部＝**無變形**
   - 臉裁切框反推公式（差 0.3 驗證過）：全身立繪 `[left0%, top7.2917%, w100%, h75%]` → resize 512² ＝ 餵模型的臉。**換新長相圖就照這公式重裁**（見下「換角色長相 SOP」）
3. **撥號前（等待層）＝打招呼→待機影片**
   - `FACE_MOTION[template]={hello,idles[]}`＋`FaceIdle` 引擎（雙播放器交叉淡入、**換片點不黑閃**、本就設計好）
   - 影片規格：1080×1916、24fps、~5s、頭尾無黑格、loop 接點順（新影片照這規格出）

**驗收＝看「App 最終合成的 9:16 畫面」**（不是模型幕後的 512 壓扁圖）。拍圖工具：`?callmock=1`(動臉層)/`?photomock=1`(照片層)＋headless Edge 截圖。

## 🕳 踩過的坑 × 根因 × 解法（不要重踩）
| 症狀 | 根因 | 解法 |
|---|---|---|
| 上下墨綠遮罩 | `.livevid img{opacity:0}` 連立繪一起藏、露深綠底 | 排除 `.fh-bg`、立繪永顯 |
| 臉「被壓扁」 | 給 Edward 看模型 512 幕後壓扁圖 | 只給 App 最終 9:16 合成圖驗 |
| 版本鬼打牆(封面新內頁舊) | iOS 外殼版號 bump 但 web bundle 沒 cap sync | 內頁印章鐵證＋cap sync 必跑 |
| 改了看不到 | 快取（無 no-cache＋批號沒換） | no-cache＋?v= 批號＋cap sync |
| 解析度低/糊 | 512 模型輸出天花板、放大2-3倍 | 銳化=假利(Edward否決)；真解=1024(Pro模型10.8fps不即時/或超解析)——研究中 |
| 講話斷斷續續 | 餵料卡即時節奏、無存貨墊底 | 拔即時閘門、囤1.5s存貨(MAX_AHEAD_S) |

| 嘴快聲慢 | 舊校時器/faceSyncMs 在新引擎仍動手 | flashhead 下禁用校時器、faceSyncMs 寫死 |
