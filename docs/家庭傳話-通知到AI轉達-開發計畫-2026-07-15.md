## 1. 產品原則

1. **通知只負責叫人回來**：鎖定畫面預設顯示「媽媽請沐寧帶了一句話給你」，不放完整內容。
2. **App 內才完整轉達**：首頁顯示待聽卡，提供「聽沐寧說」與「看文字」。不可一開首頁就自動出聲。
3. **原文是唯一真相**：AI 可加一句短開場或關心，但家人原文必須以引號完整轉達，不得自行改寫原因、語氣或結論。
4. **不把推播當送達證明**：APNs 接受請求只算「已發送」，AI 完整說完才算「已聽」；看文字、回覆及檢舉分開記錄。
5. **沒有真訊息就不顯示傳話**：移除目前首頁與開場中的示範 fallback，不得捏造「美華說……」等家人內容。
6. **隱私優先**：使用者可自行開啟通知預覽；健康、用藥、金錢與安全相關內容即使開預覽，第一版仍只顯示泛化提醒。

## 2. 完整使用流程

### 2.1 發送端

1. 使用者對 AI 說：「跟小宇說，晚上早點睡。」
2. AI 必須複誦：「你要我跟小宇說『晚上早點睡』，對嗎？」
3. 使用者確認後才建立訊息；建立成功後 AI 才說「好，我會帶給他」。
4. App 顯示可追蹤狀態：已送出／已通知／已看文字／已聽／已回覆。

### 2.2 收件端通知

預設通知：

- 標題：`媽媽請沐寧帶話給你`
- 內容：`打開沐寧，讓沐寧說給你聽`
- 深連結：傳話 ID，不帶原文。

通知送達後，不將訊息標為已讀或已聽。APNs 失敗時留在待重試佇列，訊息本身仍可在 App 內取得。

### 2.3 首頁待聽卡

- 標題：`媽媽有話請沐寧帶給你`
- 副文：`今天 20:15`，預設不露原文。
- 主要按鈕：`聽沐寧說`
- 次要按鈕：`看文字`
- 更多動作：`稍後提醒`、`移除`、`檢舉`

「聽沐寧說」直接進聊聊並帶入指定 relay ID；「看文字」開底頁顯示傳話者、原文與時間，仍保留之後再聽的能力。

### 2.4 聊聊轉達

1. 語音與 Avatar 都 ready 後才開始。
2. 指定 relay 經後端 claim 與簽章驗證後，AI 先說：`小宇，媽媽要我跟你說：「晚上早點睡。」`
3. 最多加一句短關心，不推測家人沒說的理由。
4. 完整播畢才 ack 為已聽；斷線、插話或初始化失敗必須 release，下一通可重播。
5. 說完可問：`要不要請我回媽媽一句？` 回覆會建立一則新的傳話，不覆寫原訊息。

## 3. 資料與狀態契約

PR #42 的 `family_relay_messages` 保留作原文與收件人 SSOT；`delivered` 僅代表 AI 完整轉達，不代表推播送到。

第一版新增獨立時間戳／receipt：

| 欄位／事件 | 意義 |
|---|---|
| `push_requested_at` | 已排入推播佇列 |
| `push_accepted_at` | APNs 接受發送，不保證使用者看到 |
| `opened_at` | 使用者由通知或首頁開啟傳話 |
| `text_viewed_at` | 已看原文，仍可再聽 |
| `delivered_at` | AI 已完整說完 |
| `acknowledged_at` | 使用者按知道了或口頭確認 |
| `reply_relay_id` | 回覆建立的新傳話 ID |

終止狀態維持 `cancelled`、`reported`、`expired`。同一 relay 的推播、文字閱讀與語音播放必須可冪等重試。

## 4. 技術施工範圍

### 4.1 既有 PR #42 先完成

- 指定收件人建立／claim／ack／release／report。
- 同家庭成員驗證、RLS、Brain-to-Voice 簽章。
- AI 傳話前確認原文；語音提醒動作必須等 App 成功回執。
- 移除 JSON fallback 在 production auth 模式下的靜默成功，正式環境缺表或缺密鑰要 fail closed。

### 4.2 APNs 與裝置登錄

新增 `push_devices`：person、APNs token、sandbox/production、app version、locale、啟用狀態、last seen、invalidated time。Token 僅供後端服務角色使用，不開放家庭成員讀取。

新增 durable outbox：relay 建立與 outbox 寫入同一交易；worker 對每個有效裝置送 APNs，支援指數退避、無效 token 停用、去重 key 與錯誤紀錄。不得把完整原文放入預設 payload。

### 4.3 iOS／Capacitor

- 加 Push Notifications capability 與 production entitlement。
- 註冊／更新／失效 device token，登入、登出與換帳號時同步歸屬。
- 接收前景通知及點擊事件，依 relay ID 導到首頁待聽卡、文字底頁或聊聊。
- 通知設定頁顯示已開／未開；被拒絕時引導到 iOS 設定。
- 不申請 Critical Alerts 作為第一版前置，也不宣稱可突破靜音。

### 4.4 App 內 UI

- 新增真 relay loader，不再從 `familyFeed2` 字串猜測傳話。
- 首頁招呼不直接顯示完整原文；以待聽卡取代示範內容。
- `聽沐寧說` 將 relay ID 帶入通話，不讓不同裝置競領錯訊息。
- 補文字底頁、再聽一次、稍後提醒、回覆、移除與檢舉。
- 既有 UGC 檢舉機制改送 relay ID 與伺服器保存的原文雜湊，避免只記本機字串。

## 6. 上線驗收門檻

1. A 傳話給 B，C 不可讀取、claim 或收到推播。
2. B 在 App 關閉時收到泛化通知；鎖定畫面不露原文。
3. 點通知打開正確 relay，首頁不自動出聲。
4. 看文字不會讓 AI 永久跳過；仍可按「聽沐寧說」。
5. AI 清楚說出傳話者及原文，不增刪意思。
6. 播放完成只 ack 一次；斷線 release；強制關閉後可安全重試且不連續重播。
7. 回覆建立新 relay，原訊息與回覆可追蹤。
8. 登出、退出家庭圈、刪除帳號或 APNs token 失效後不再收到該家庭推播。
9. 通知被拒絕時 App 有明確狀態與設定引導；功能仍可從 App 內待聽清單使用。
10. 後端可看 queued／accepted／failed／opened／heard，但不得把 APNs accepted 說成已送達使用者。
