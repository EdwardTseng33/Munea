# 沐寧 · 免登入（訪客）模式能用什麼 · 設計建議（2026-07-09 · Edward 點題）

## 一、現況（程式實查）
- **有免登入 onboarding**：`web/onboarding.html` 選角色＋取名＋同意 → `index.html`，全程不強制登入/註冊。
- **登入目前可有可無**：`app.js` 從無「登入牆」；不登入＝訪客模式，功能照用。
- **把關是看「方案」不是看「登入」**：家人邀請/入圈等擋的是 `MMPLAN.isFree()`（免費方案），跟有沒有帳號無關。
- **後端已支援無帳號試營運**：`/account-bootstrap` 收到沒帳號會回 `auth_user_required` → 標 `pending-auth`、照樣進；家庭資料用「裝置編號」記帳，真帳號上線自動回雲端桌子。

→ 結論：**帳號狀態下能用什麼，目前沒設計過**。以下是建議。

## 二、設計原則
長輩對「註冊/登入」極度怕麻煩——**強迫登入＝第一關就流失**。所以：
> **一個人自己用的功能，全部不用登入；牽涉「別人」或「別的手機」的功能，才在用到的當下請他登入。**

## 三、建議：兩類功能

### 🟢 訪客就能用（免登入、本機為主，等於免費方案的完整體驗）
| 功能 | 為什麼不用登入 |
|---|---|
| 跟寧寧聊天（語音＋會動的臉）| 核心價值、不涉他人；免費 5 分鐘/次照舊 |
| 狀態頁（自己的血壓/心率/步數，接 Apple 健康）| 資料在自己手機、本機顯示 |
| 心情記錄（情緒球）| 本機 |
| 吃藥/回診提醒（App 關著也響）| 本機通知、不需雲端 |
| 家庭活動、記錄簿（自己這台）| 本機；只有「跨裝置同步」才需帳號 |
| 買 Plus/Pro、加值點數 | 蘋果付款綁 Apple ID、不需 App 登入即可付＋恢復購買 |

### 🔵 要登入才能用（本質需要「雲端身分」＝跨人/跨裝置）
| 功能 | 為什麼一定要登入 |
|---|---|
| 邀請家人、加入照護圈 | 要把兩支手機綁成同一家＝需要真帳號當「身分證」 |
| 家人看到我的健康數據 / 我看家人的 | 跨裝置同步、要認得「是同一家人」 |
| 傳話（幫我跟○○說…） | 送到對方手機＝需要雲端身分 |
| 記憶跨手機接續（換手機不失憶）| 記憶要認得「這是同一個人」 |
| 資料匯出 / 刪除帳號（法遵）| 要確認身分才能給/刪個資 |

**做法＝「用到才問」**：訪客按「邀請家人」時，跳一句「要連上家人，先用 Google 或 Apple 登入一下（30 秒）」——**不是一開 App 就擋登入牆**。

## 四、方案 × 登入 兩軸釐清（避免打架）
現在只有「方案」一軸，建議補「登入」為第二軸：
- **訪客 × 免費**：完整 solo 體驗（聊天/健康/提醒/心情）。
- **訪客 × 買了 Plus/Pro**：solo 體驗更足（點數多/歷史長），但**家人功能仍要先登入才給**（因為家人本質需雲端身分）。→ 買 Plus 後溫和提示：「登入就能邀家人＋換手機不失憶」。
- **登入 × 任何方案**：家人/跨裝置/法遵功能開通，受方案人數上限規範（Plus 4 人 / Pro 12 人）。
