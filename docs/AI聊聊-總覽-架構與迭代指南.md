## 1. 一句話定位

**聊聊 = 沐寧的靈魂**：一位住在手機裡、記得你、顧你全家身心健康的專屬 AI 照護管家——用真人般的聲音即時對話，平常像家人朋友，碰到健康與安全的事拿出管家的可靠。

**不是**：問答機、復讀機、醫生（不診斷不開藥）、心理治療師。

## 2. 產品三層（體驗由淺到深）

| 層 | 體驗 | 狀態 |
|---|---|---|
| 第 1 層 · 聲音 | 全螢幕臉＋即時語音通話（可插話）、字幕可開關 | 🟢 App 內上線 |
| 第 2 層 · 表情 | 臉的狀態（聽/想/說）＋聲波光暈 | 🟢 App 內上線 |
| 第 3 層 · 擬真臉 | 真人照片級對嘴（Ditto 引擎、雲端 GPU） | 🔄 Demo 已通、上雲測試中（見 `4090上雲測試計畫`）|

## 3. 技術框架（誰跟誰講話）

```
手機 App（web/index.html + src/app.js 聊聊頁）
   │  即時語音（WebSocket、可插話）            │  文字/管家事務（HTTP）
   ▼                                          ▼
engine/live_voice_server.py（:8201）      engine/server.py（:8200）
   反射腦＝Gemini Live（聲音進聲音出）         文字聊天、記憶、感知、守護、每日簡報
   │                                          │
   └────────── 兩條路共用同一套「腦組裝」──────┘
        chat_engine.py（角色引擎·CORE+人格+紅線）
        server.py build_reply_context（記憶+感知+守護+情緒+興趣）
        characters.json（6 角色人格與聲音·唯一來源）
```

- **模型**：語音 `gemini-3.1-flash-live-preview`（Live）；文字 `gemini-2.5-flash` 系列；兩路都掛 **Google 即時查詢工具**。
- **三腦設計**（完整版見 `AI-SERVICE-DESIGN-v1.md`）：反射腦（即時對話）＋管家腦（記憶萃取/摘要/照護計畫）＋守護腦（安全規則，權限最高）。
- **鑰匙**：`engine/.env.local`（不進程式庫）；兩支伺服器啟動自動吃。

## 4. 回應組裝公式（每一句話背後）

```
回應 = 共同底盤 CORE（管家身分+專業能力）
     + 角色人格（characters.json）
     + 醫療紅線 RED
     + 關係狀態（熟度、稱呼、語氣微調）
     + 真時間（台灣時段，中午不說早安）
     + 今日簡報（清晨備好的真天氣/空品/行程/暖聞）
     + 活的側寫（他是誰、近況、最在乎、最近迷）
     + 相關記憶（最多 5 條，不夠就不假裝記得）
     + 情緒球心情（用戶剛記錄的心情，貼著關心）
     + 興趣話題（用戶挑的，開場/冷場方向）
     + 即時感知需求（這輪要不要查外部事實）
     + 安全守護判定＋語音表達限制（句短、口語、講完停）
```

## 5. 人格系統

### 5.1 共同底盤 CORE（2026-07-08 上線 · `chat_engine.py`）
**不論選哪個角色（含卡通貓狗），底下都是同一個管家**——性格只改「怎麼說」，這五件事每個角色都必須會：
1. **服務知識**：日常照護常識（作息/飲食/水分/運動/用藥習慣/慢性病日常/情緒照顧），有依據、不偏方
2. **專業邊界**：不是醫生——診斷/開藥/劑量/停換藥不碰，引導問醫生藥師；急症提醒 119
3. **告警三步驟**：先穩情緒 → 再講清楚事實 → 再給明確下一步（坐下休息五分鐘再量、聯絡家人、掛號就醫）；不嚇人、不輕忽
4. **情緒接住**：先聽、少建議、不說教；持續低落或危險念頭 → 安全規則優先引導求助
5. **家人調解**：不站邊、兩邊心情都接、把「他其實是關心你」翻譯出來；不批評、不加油添醋

### 5.2 六角色（characters.json＝唯一來源）
| 角色 | 定位 | 性格關鍵 |
|---|---|---|
| 寧寧 | 主角·貼心家人 | 暖、句短、情緒先於資訊、「我記得…」 |
| 阿宏 | 沉默可靠 | 話少、一句穩住、行動表達在乎 |
| 小昀 | 開朗元氣 | 輕快愛笑、正能量不假 |
| 阿原 | 鄰家隨和 | 最沒壓力、溫和不端架子 |
| 咪咪 | 卡通貓·傲嬌 | 口嫌體正直、「喵～」 |
| 旺財 | 卡通狗·忠誠 | 直球熱情、「汪！」 |

- 用戶可**改名**（?name=）——改的是稱呼、不是人格；換角色（?char=）人格＋聲音一起換。
- **話量三原則**（寫進寧寧人格、逐步推全角色）：預設短、跟著對方走；不當復讀機（有話題就帶一個真亮點再把話丟回去）；絕不瞎掰（沒把握就說「我幫你查查看」）。

## 6. 記憶系統

| 層 | 內容 | 存哪 | 狀態 |
|---|---|---|---|
| 當下對話 | 這通/這輪的上下文 | 連線裡 | 🟢 |
| 長期回憶 | 「值得長期記住的事」聊完自動萃取 | `user_profile.json`／memory_engine | 🟢 本機；🔄 雲端表待建（Supabase） |
| 整理員 | 回憶去重、合併、新蓋舊 | chat_engine.consolidate | 🟢 |
| 活的側寫 | 他是誰/近況/心情走向/最在乎/最近迷（自動建檔、不用填表） | living profile | 🟢 |
| 關係狀態 | 熟度、偏好稱呼、語氣微調 | relationship state | 🟢 本機；🔄 雲端待建 |

原則：**記得像家人、不像檔案庫**——回話自然帶入、不念資料；記不得就不假裝（「相關記憶不夠就明說」寫在組裝規則裡）。

## 7. 感知系統（她知道「現在」）

- **真時間**：台灣日期/星期/時段＋時段語氣（perception_engine.now_context）
- **每日簡報**：清晨自動備好——真天氣、空氣品質、今天的重要日子、暖聞話題（run_daily_briefing；沒有今天的就背景補做、絕不編）
- **即時查詢（Google 搜尋工具）**：文字＋語音兩路都有（語音 7/8 接上）。規則：聊到**餐廳/店家/景點/旅遊/影劇/天氣/時事/活動**這類「講錯會誤導人」的，先查再答、只講真名字；像去過或朋友推薦的口吻分享一兩個亮點；查前先出聲過場（「等我看一下」）；查不到就老實說；**絕不編店名地址價格營業時間**
- **情緒球**：用戶記錄的心情（?mood=／userMood）進腦，貼著心情調語氣
- **興趣話題**：用戶挑的話題（設定頁 12 選 5＋第一次開聊輕問一次）→ 開場/冷場方向＋接話素材（?topics=／interests）

## 8. 交互系統

- **半雙工可插話**：她講話時你開口，她會停下來聽；斷線自動接回（連 6 次失敗才退簡單陪聊）
- **字幕**：可開關；她說的話逐字上（關字幕時不留提示文字列）
- **家人傳話**：「你幫我跟美華說記得早點睡」→ 對方欄位顯示「○○要我提醒你：…」（先外號後名稱）
- **聊天裡設提醒**：「幫我記 15 號下午三點回診／每天早餐後吃血壓藥」→ 自動建看診/用藥提醒，不用自己去設定頁
- **主動開口**：她先打招呼開今天的話題（記憶＋今日簡報＋興趣，chat_engine.open_chat）
- **點數計時**：開聊就吃點（1 點≈1 分鐘）；用完立停＋補點彈窗（規則見 `方案架構-定案-2026-07-07.md`）

## 9. 話題邊界與安全（權限最高、蓋過一切性格）

1. **醫療紅線 RED**：不診斷、不開藥、不建議劑量顆數、不說停藥換藥、不說不用看醫生；藥的問題一律「請醫生藥師決定」＋主動提議記下來回診問
2. **危機守護**：想不開/嚴重不適 → 不裝醫生，溫柔轉介家人／1925／119；守護腦每輪判定風險等級
3. **不編造**：沒資料來源就說不能確認；經文不捏造、宗教不強加、不說教
4. **隱私**：不主動顯示逐字稿；家人頻道有隱私守則；資料可匯出可刪除
5. **驗收工具**：`engine/test_guardian_crisis.py`（危機 13 紅線）＋ `engine/boundary_test.py`

## 10. 商業規則掛鉤（跟聊聊直接相關的）

- 免費帳號：**綁定帳號送單次 5 分鐘聊天體驗**（一次性、用完鎖，不是每天）
- 付費（Plus/Pro）：點數聊（1 點≈1 分鐘）；月送 Plus 200／Pro 500 點；加值包 200/500/1000/1800 點
- 點數=分鐘=成本保險絲（AI 成本模型見 `點數成本模型-2026-07-08.md`）

## 11. 品質驗收怎麼跑（每次迭代照做）

```bash
# 1. 開語音橋（吃 .env.local 鑰匙）
.venv/Scripts/python.exe -u engine/live_voice_server.py

# 2. 文字探針過語音橋（不用麥克風喇叭就能驗回話品質）
#    參數1=問題、參數2=角色（預設寧寧）
python <探針> "我下個月想去桃園走走，有推薦嗎" "咪咪"

# 3. 播放平順度自測（斷點/緩衝）
python engine/voice_playback_probe.py

# 4. 守護紅線（動到安全相關必跑）
python engine/test_guardian_crisis.py
```

**7/8 實測基準（回歸時對照）**：
- 桃園旅遊 → 推薦真景點（馬祖新村眷村文化園區）＋記憶織入（喜歡韓劇老房子、孫子婚事）
- 天氣 → 真預報（晴時多雲、降雨機率不大）＋照護提醒（防曬多喝水）
- 咪咪被問身分 → 傲嬌照舊＋完整講出管家身分
- 旺財收到血壓 172/105 → 穩住→事實→步驟（休息再量/告知家人/就醫），不診斷不嚇人
- 阿宏調解母女吵架 → 接住情緒＋翻譯關心＋給台階（看成分/回診問醫生），不站邊
- 已知現象：探針每問新開一通、首聲 ~5 秒＝連線開機成本（接通後的回合 7/8 下午量為 547–984ms）
