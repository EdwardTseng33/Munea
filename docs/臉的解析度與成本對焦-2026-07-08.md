## 1. 手機上的解析度：目標規格定錨

**長輩的參照物不是電影，是「跟兒女視訊」。**

| 事實 | 數字 |
|---|---|
| 長輩常用機（舊 iPhone / SE / 中階安卓）螢幕 | 750×1334 ～ 1080×2340 |
| LINE / FaceTime 視訊實際畫質 | 約 480p～720p |
| HeyGen LiveAvatar 即時檔規格 | 720p（最高檔才 1080p）|
| Tavus 即時檔 | 同級（720p 檔位）|

→ **產品目標規格定錨：直式 720×1280、每秒 25–30 幀、傳輸 4–8Mbps。**
做到這個＝「跟真人視訊同級」＝遠景三家同一個檔位。**不需要、也不該追 1080p+**（成本翻倍、長輩手機與眼睛都感受不到差異）。

**現況（7/8 demo）**：715×960、25 幀已跑通——整體解析度已在門檻邊，缺口在「嘴部細節」（Wav2Lip 引擎畫嘴的畫布只有 96px）＝**引擎等級問題，不是解析度問題**。

## 3. 499 / 999 撐不撐得住：毛利帳

**關鍵結構：點數＝分鐘（1 點=1 分鐘）＝成本上限是「產品規則保證的」**——用戶最多吃掉「送的點＋買的點」，變動成本天生封頂；加值點數本身另有毛利。

| | Plus 499 | Pro 999 |
|---|---|---|
| 實收（Apple 15% 後）| 424 | 849 |
| 臉成本上限（點全用完 · 1路保守）| 72 | 180 |
| 語音腦成本上限（Gemini Live · 待卡西法實測 benchmark）| 估 40–90 | 估 100–225 |
| **毛利（最壞情況：點數 100% 用好用滿）** | **~62–74%** | **~52–67%** |

- 實務上月點數用滿率通常遠低於 100% → 實際毛利更好。
- 尖峰同時在線由「按需開卡、沒人看就關」控制（scale-to-zero、顯卡經濟學文件既定策略）。
- ⚠️ 語音腦那格是估算，上 RunPod 驗證時一起實測回填。

**結論：499/999 不必動。天花板由點數模型鎖住，臉的成本佔實收 <2 成。**

## 4. 怎麼到「遠景效果」：達標路徑（照既定框架、無新發明）

遠景效果拆解＝ ①臉會講話（對嘴）②臉是活的（眨眼/表情/頭微動）③接得快（<1秒）④ 720p 視訊感。

| 階段 | 內容 | 對應缺口 |
|---|---|---|
| **P1（已完成 7/8）** | 本機 demo：語音鏈 × 對嘴 × 串流管線全通（715×960@25） | 證明管線；④半達標 |
| **P2（下一步 · 幾天+幾美元）** | RunPod 4090 + **Ditto** PoC：全臉驅動（眨眼/表情/頭動跟聲音走）、720p benchmark、單卡幾路實測 | ②③④——Ditto 是全臉引擎，直接解「只動嘴」的不自然 |
| **P3（上線版）** | scale-to-zero 派卡＋併發池、畫質強化層視 benchmark 選配 | 尖峰成本＋嘴部細節餘量 |
| **對照組（隨時可開）** | Tavus 免費 25 分／$59 百分鐘：建寧寧專屬人物（US$40–65 一次性）當「遠景效果對照組」與提案展示 | 給評分基準、不進量產 |

**人物識別鐵律**：全程用寧寧官方形象（一張照片可建 Tavus 人物；Ditto 亦以官方素材驅動）。臉＝產品識別，任何引擎替換都不得換臉。
