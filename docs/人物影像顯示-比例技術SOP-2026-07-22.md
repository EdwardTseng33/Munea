# 人物影像顯示 · 比例技術 SOP

**建立**：2026-07-22（B2B Demo 聊聊臉被拉長事故後，Edward 指定立此文件確保不再重犯）
**一句話鐵律**：產品裡任何人物照片／臉，顯示一律「**維持比例、絕不拉伸**」。寧可裁掉一點邊，也不能把臉壓寬或拉長。

---

## 為什麼立這份（2026-07-22 事故）

B2B Demo 聊聊的臉被縱向拉長變瘦、比例失真。根因是兩件事疊加：

1. **臉層用「拉伸填滿」貼**：`.overlay video` 用 `object-fit: fill`，把 768 正方形影片硬撐成臉框的形狀。
2. **臉框本身變成長方形**：手機直式畫框（`.frame`）同時寫死 `width` 和 `height:100dvh`，導致 CSS 的 `aspect-ratio:1080/1920` 被忽略、失效，畫框走樣成非 1080:1920，臉框（框內百分比定位）跟著變長方形。

兩者疊加 → 正方形的臉被塞進長方形框、又用拉伸貼 → 臉變形。量測結果：臉框實際長寬比 0.82（該是 1.0 正方形），所有手機寬度都拉伸。

---

## 三條技術鐵律

### 鐵律 1：顯示端一律 `cover`，禁用 `fill`
- `object-fit: cover` = 維持比例、填滿容器、裁掉超出部分 → **臉永不變形**
- `object-fit: fill` = 硬撐填滿 → **臉會變形**，禁用於任何人物影像
- **基準**：App 正式線 `web/demo.html` 的 `.face-bg img` 就是 `object-fit:cover; object-position:center 18%`。所有 Demo／展示的臉層必須跟它一致。

### 鐵律 2：模型端 fit ≠ 顯示端 object-fit（不可混用）
- 條件圖規格裡的 `fit`（`demo_profile.py` 寫 `fill`）指的是「模型輸入怎麼縮圖」，跟畫面怎麼貼**無關**。
- 顯示端 object-fit 是「畫面上臉怎麼貼」，一律 `cover`。
- 本次 bug 正是把條件圖的 `contract.fit`（模型端 = fill）直接拿去設 `faceVid.style.objectFit`（顯示端）→ 語意誤用。**顯示端 object-fit 不可跟著條件圖的 fit 走。**

### 鐵律 3：容器比例別靠「同時鎖死 width + height」
- CSS 規則：`width` 和 `height` 都給明確值時，`aspect-ratio` 會被忽略、形同沒寫。
- 要維持比例：用 `aspect-ratio` + 只給一個維度（另一個留 auto），或用 `calc()` 把高綁死在寬上。
- 本次 `.frame{width:var(--frame-w);height:100dvh;aspect-ratio:1080/1920}` 三者並存 → aspect-ratio 失效。

---

## 驗證清單（改任何人物顯示後必跑，一條都不能省）

1. **不信設定數字，量畫面實際框**：撥通後量臉層容器的實際像素長寬比（`getBoundingClientRect`），對照它「該有的比例」。設定寫「正方形」不代表畫面上真的是正方形。
2. **撥通截圖看臉**：臉型自然、不瘦長不壓寬、跟背景身體銜接順、無疊影、無錯位。
3. **對照 App 正式線**：貼法（cover + object-position）與 App 一致。
4. **多種手機寬度都量**：390 / 414 / 430 都要（本次 bug 在每種寬度都拉伸，單測一種會漏）。

> **歷史教訓**：這類「臉變形」肉眼一看就有感，但看程式設定的數字常被騙（設定寫正方形、畫面卻是長方形）。**最終判準永遠是撥通後量畫面 + 真人看截圖**，不是讀設定。
