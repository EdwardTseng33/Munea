> 原則：**借鏡 2026 最成熟/最紅的做法、不重造輪子**，並貼合沐寧（長輩、中文台灣、非醫療、子女買）。

# 一、記憶層（讓寧寧「真的記得長輩」）

## 借鏡誰（2026 SOTA）
- **Mem0**：消費級 App 首選；向量抽取＋計分檢索；**寫入不覆蓋、留多版本、檢索時定當下相關**；開源可自架。
- **Zep / Graphiti**：時序知識圖譜＋**事實有效期窗**（fact validity window）；適合「狀況會變」的場景。
- **Nomi AI（最貼近我們）**：**四層記憶 + Identity Core**（沉澱長輩核心自我、讀弦外之音、主動追問）。
- **Replika 2.0**：**記憶儀表板**讓用戶查看/修正記憶。

## 我們的設計（分四層 + 底層接成熟引擎）
1. **Identity Core（核心自我 · 永遠在場）**：長輩的名字/稱呼、家人與關係、重大事件（喪偶、搬家）、價值觀、禁忌話題。穩定、少量、每次都注入；由管家腦持續增修。
2. **長期記憶（可檢索）**：偏好、關係細節、生活事件、健康情境。**底層接 Mem0（或 Zep 補時序）**做語意向量召回＋衝突處理＋衰減；上層維持我們的 `memory_items` 業務欄位（type/content/confidence/importance/sensitivity）。
3. **短期/當次**：本通對話脈絡。
4. **主動追問回路**：管家腦聊完（post-turn）萃取新記憶＋標記「值得回頭關心」（例：孫子下月結婚 → 下次主動問「婚禮順利嗎？」）。這是 Nomi 的核心賣點、也是我們「主動關心」的燃料。

## 三個一定要補的機制（現在缺）
- **語意召回**：能把「幾個月前提過一次的細節」找回來（純計分不夠）。
- **時序 + 衝突**：每筆記憶帶「有效期/時間」；狀況變了（搬家/住院/過世）**保留多版本、回話時用當下版本**，不「自信說錯」。
- **遺忘衰減**：低重要性/過期的 temporary_event 自動淡出，維持召回品質與成本。

## 收斂雙軌（先解最痛）
- 現況兩套：`user_profile.json`（舊·中文 key·手工）vs `memory_items.json`（新·schema·AI 萃取）。
- 做法：**以 memory_items 為唯一真相**，把 user_profile 的「回憶/喜好」一次性搬進 memory_items + Identity Core；`chat_engine` 改讀同一套；舊檔退役。

## 子女信任（對「子女買」）
- **記憶儀表板**（抄 Replika）：子女可看/修正長輩的記憶——但**受同意分層 RLS 控管**（見子女授權設計），原始私密對話不攤開。

## 搭建步驟
1. 收斂成一套（user_profile → memory_items + Identity Core）＋驗證聊完真的寫回。
2. 底層接 Mem0：語意召回 → 事實有效期 → 遺忘衰減（照此序）。
3. 管家腦 post-turn 主動追問回路。
4. 子女記憶儀表板（接同意分層）。

# 二、感知層（讓寧寧「懂長輩此刻的處境」）

## 借鏡誰（2026 SOTA）
- **工具呼叫 / MCP grounding**：一個標準接口掛多個「即時事實來源」；**Google Maps Grounding（地點/天氣/路程）**、Search Grounding。
- **核心原則**：**精準、有範圍地投餵**（只在需要時給最小必要事實，不爆 token）。

## 我們的設計（把「感知計畫」落成真的工具）
現況已有對的抽象：`topic_perception_plan`（對話主題 → 需要哪些感知源）。缺的是把它接成**真的工具呼叫**：
- **時間/日期/節慶/回診**：本機時間 + 行事曆 + 記憶（回診日）。← 最簡單、現在就能接。
- **天氣**：**中央氣象署 CWA 開放資料 API**（台灣權威）或 Google Maps 天氣 grounding。
- **在地/交通/就醫**：**Google Maps Grounding（MCP）** 錨定地點、營業時間、路程。
- **個人近況**：直接讀**記憶層**（把感知與記憶打通）。

## 運作方式
1. 管家腦看 `topic_perception_plan` → 判斷這輪需要哪些源。
2. 只呼叫需要的工具（用 Gemini Live 現成的 function calling）。
3. 把結果**最小、scoped** 注入即時語音的當下脈絡（不整包塞）。
4. 結果快取進 `perception_snapshots`（帶 expiresAt），避免重複呼叫。

## 感知 × 安全 × 主動（打通三腦）
- **餵守護腦**：寒流/高溫/空品差 → 主動關懷提醒（「今天很冷，膝蓋要保暖喔」）。
- **餵主動開口**：感知 + 記憶 → 主動關心（下雨且長輩愛散步 → 「今天下雨，改在家做點伸展好不好？」）。

## 搭建步驟
1. 接「時間/日期」→ 立刻擺脫寫死。
2. 接「天氣」（CWA）→ 真天氣取代 hardcoded「轉涼有寒流」。
3. 接「在地/交通/就醫」（Maps grounding）。
4. 感知結果接進管家腦（主動關心）＋守護腦（安全預警）。
